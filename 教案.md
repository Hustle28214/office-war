# 教案：《明天能上线吗》

> **定位：** Unity + C# 游戏开发 **跟做练习**  
> **用法：** 严格按 **课 01 → 课 20** 顺序；打开当课「涉及文件」，读文件头注释，填空实现，Play 自测。  
> **搜索：** IDE 全局搜 `课 XX` 或 `LEARN:` 定位当课代码。  
> **答案：** [`Reference/`](Reference/)（结构参考；协作版枚举/胜负以本教案 + `plan.md` 为准）  
> **规则：** [`plan.md`](plan.md)（机制）· 本教案（步骤）

**设计一句话：** 协作把项目做成（德式多轨 KPI）· 回合=工作日 · 类 Kards 三行 · 职位卡包 · 钉钉感 UI · 出牌要有甩文档打击感。

## 一、开始前

### 1.1 环境

- Unity **2022.3 LTS+**
- 打开项目根目录 `office-war`
- 场景：`Assets/_Project/Scenes/Battle.unity`
- 菜单：`明天能上线吗 / Setup Demo Scene`

### 1.3 每课固定流程

1. 读教案「本课步骤」  
2. 打开「涉及文件」，读文件顶部 `课 XX` 注释  
3. 实现代码（先写逻辑，再 Play）

5. `git commit -m "课XX: …"`（可选但推荐）

## 课 01｜熟悉项目与 Demo 目标

**学习目标：** 知道最终要做什么，本课不写代码。

**步骤：**

1. 阅读 `plan.md` §一（钉钉美术）、§二（协作胜负 / 工作日 / KPI）、§三.0（职位与卡包）  
2. 浏览 `Assets/_Project/Scripts/Battle/BattleManager.cs` 文件头，看组合了哪些模块  
3. 画出三块区域：**压力后方 CrisisBack / 前线 Frontline / 项目组后方 TeamBack**  

**自测：** 能说明「怎样算上线成功」「咖啡干什么用」「职位和大类卡包是什么关系」。

## 课 02｜Core：枚举与常量

**涉及文件：**

- `Core/GameEnums.cs`
- `Core/GameConstants.cs`

**学习目标：** 用枚举表达协作局状态；魔法数字进常量。

**步骤：**

1. `GameEnums` 补全：`JobType`、`Faction(Team/Crisis)`、`BoardRow`、`TurnPhase(MorningStandup/Main/DayEnd)`、`GameResult(Ongoing/TeamWin/TeamLose)`、`PlayerRole`、`RoleArchetype`、`KpiTrack`、`CardFeel`、`PlayerActionMode`  
2. `GameConstants`：Sprint 天数、咖啡、CrisisBudget、KPI 达标线、连锁上限（见文件内 LEARN）  
3. 编译无报错  

**自测：**

- [ ] 枚举命名与 `plan.md` §五.2 / §二.3 一致  
- [ ] 没有 `Player`/`Enemy`/`PlayerWin` 旧名  

**参考：** `Reference/Core/`（若仍是旧名，以本课 LEARN 为准）

## 课 03｜Data：卡牌与特色卡包

**涉及文件：**

- `Data/OfficeCardData.cs`
- `Data/CardPackData.cs`
- `Data/CardCatalog.cs`

**学习目标：** SO 配牌；`Role → Archetype → Pack`；入门用 Starter 拼库。

**步骤：**

1. `OfficeCardData`：费用、精神值、kpi、`CardFeel`、可选 `exclusiveRole`  
2. `CardPackData`：`packId`、`role`、`archetype`、`cards[]`、`isStarter`  
3. `CardCatalog.CreateStarterDeck(role, archetype)`：核心包 + Starter 子路线包  

**自测：**

- [ ] Demo 默认 `Dev + DevFrontend` 能拼出 ≥ 20 张（或 plan 建议的 20~24）  

**参考：** `Reference/Data/` + `plan.md` §三.0

## 课 04｜Cards：卡组与手牌

**涉及文件：**

- `Cards/RuntimeCard.cs`
- `Cards/Deck.cs`
- `Cards/Hand.cs`

**学习目标：** 运行时实例 vs 配置；洗牌与抽牌。

**步骤：**

1. `RuntimeCard` 持有 `OfficeCardData` + `InstanceId`  
2. `Deck.Load` + Fisher-Yates `Shuffle` + `Draw`  
3. `Hand`：`Add` / `Remove` / `MaxHandSize`  

**自测：**

- [ ] 两次洗牌抽牌顺序大概率不同  
- [ ] 手牌满时 `Add` 无效  

---

## 课 05｜Resources：咖啡池 + KPI 看板

**涉及文件：**

- `Resources/DepartmentState.cs`
- `Resources/KpiBoardState.cs`

**学习目标：** Team 共享咖啡；Crisis 有 `CrisisBudget`；胜负看 KPI 轨。

**步骤：**

1. `DepartmentState`：`TrySpendCoffee` / `RefillCoffee` / `IncreaseCoffeeMax`；Crisis 侧 `TakeCrisisDamage`  
2. `KpiBoardState`：五轨加减；`MeetsAllTargets()` / `HasFailed()`  
3. 变更触发 `OnChanged`  

**自测：**

- [ ] 咖啡不足不扣费  
- [ ] Delivery 提到达标线后 `MeetsAllTargets` 在其它轨未达标时仍为 false  
- [ ] Morale=0 或 Pressure 爆表 → `HasFailed`  

---

## 课 06｜Battlefield：棋盘格子

**涉及文件：**

- `Battlefield/BoardSlot.cs`
- `Battlefield/BoardController.cs`

**学习目标：** 三行棋盘 MVP（每行 1 格）。

**步骤：**

1. `BoardSlot`：`Row`、`Occupant`、`IsEmpty`  
2. `Initialize`：CrisisBack → Frontline → TeamBack  
3. `GetBackSlot(Team/Crisis)`、`FrontlineSlot`  

**自测：**

- [ ] `Slots.Count == 3`  
- [ ] `GetBackSlot(Team).Row == TeamBack`

## 课 07｜Battlefield：单位与前线

**涉及文件：**

- `Battlefield/UnitEntity.cs`
- `Battlefield/UnitFactory.cs`
- `Battlefield/FrontlineController.cs`

**学习目标：** 单位占格；前线归属 = 谁在推评审。

**步骤：**

1. `UnitEntity`：精神值、`HasActedThisTurn`、`TakeDamage`  
2. `UnitFactory.Spawn`  
3. `FrontlineController.GetOwner()`  

**自测：**

- [ ] 精神值归零触发死亡  
- [ ] 空前线 `GetOwner()` 为 null  

---

## 课 08｜Combat：清阻塞结算

**涉及文件：**

- `Combat/CombatResolver.cs`

**学习目标：** 纯静态结算，不碰 UI。

**步骤：**

1. `ResolveUnitVsUnit`：互扣 kpi  
2. `ResolveUnitVsCrisis`：只扣 `CrisisBudget`  
3. `ResolveRangedAttack`：Manager 在 TeamBack 免反击  

**自测：**

- [ ] 互殴双方掉血  
- [ ] 打 Crisis 不伤攻击者  

---

## 课 09｜Turn：工作日与 Sprint

**涉及文件：**

- `Turn/TurnManager.cs`

**学习目标：** 1 回合 = 1 工作日；Team ↔ Crisis 交替。

**步骤：**

1. `StartSprint(days)`：`ActiveFaction=Team`，`WorkDay=1`  
2. `EndDay`：切换阵营；回到 Team 时 `WorkDay++`  
3. `IsSprintOver`；事件 `OnDayStarted` / `OnDayEnded`  

**自测：**

- [ ] 连续 EndDay 四次（Team→Crisis→Team→Crisis），第二次回到 Team 时 `WorkDay == 2`  

---

## 课 10｜Battle：开局、选角、部署

**涉及文件：**

- `Battle/BattleManager.cs`（Part 1）

**学习目标：** Facade；Starter 组库；部署到 TeamBack。

**步骤：**

1. 组合 Board / Turn / Team+Crisis Department / Kpi / Hand / Chain  
2. `StartBattle`：默认 `LocalRole=Dev`、`DevFrontend` Starter；洗牌抽 3；`StartSprint`  
3. `TryPlayCardFromHand`：校验 TeamBack、扣 Hire Cost、Spawn（日志可写 CardFeel）  

**自测：**

- [ ] 部署后咖啡↓、手牌-1  

---

## 课 11｜Battle：推进、输出、连锁

**涉及文件：**

- `Battle/BattleManager.cs`（Part 2）
- `Combat/ChainResolver.cs`

**步骤：**

1. `TryAdvanceUnit` / `TryAttackUnit` / `TryAttackCrisis`  
2. 击杀或带标签部署时 `ChainResolver.TryEnqueue`（先 Log 即可）  
3. 单日连锁上限 `MaxChainPerDay`  

**自测：**

- [ ] 占线后 `GetOwner()==Team`  
- [ ] 超连锁上限有「群已禁言」类日志  

---

## 课 12｜Battle：日终、KPI 胜负

**涉及文件：**

- `Battle/BattleManager.cs`（Part 3）

**步骤：**

1. `BeginDay`：咖啡上限+1、回满、抽牌、重置 HasActed、`Chain.ResetDay`  
2. `TryHrHeal`：抬精神或推 Morale KPI  
3. `EndDay` → 切换；日终可小幅结算占线 Delivery+1（按 plan）  
4. `CheckVictory`：`Kpi.MeetsAllTargets` → TeamWin；`HasFailed` 或 Sprint 结束未达标 → TeamLose  

**自测：**

- [ ] 多工作日咖啡上限递增（≤12）  
- [ ] 人为改 KPI 达标可触发胜利（临时调试）
