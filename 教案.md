# 教案：《疯狂周一：办公室战争》

> **定位：** Unity + C# 游戏开发 **跟做练习**  
> **用法：** 严格按 **课 01 → 课 20** 顺序；打开当课「涉及文件」，读文件头注释，填空实现，Play 自测。  
> **搜索：** IDE 全局搜 `课 XX` 或 `LEARN:` 定位当课代码。  
> **答案：** 项目根目录 [`Reference/`](Reference/)（不在 Assets 内，避免重复编译）  
> **规则：** `plan.md` §三（机制）、本教案（步骤）

---

## 一、开始前

### 1.1 环境

- Unity **2022.3 LTS+**
- 打开项目根目录 `office-war`
- 场景：`Assets/_Project/Scenes/Battle.unity`

### 1.2 目录与模块地图

```
Scripts/
├── Core/           课 02 — 枚举与常量
├── Data/           课 03 — 卡牌配置
├── Cards/          课 04 — 卡组、手牌
├── Resources/      课 05 — 咖啡、HQ
├── Battlefield/    课 06~07 — 棋盘、单位、前线
├── Combat/         课 08 — 伤害结算
├── Turn/           课 09 — 回合
├── Battle/         课 10~12 — 战斗总控（分三课）
├── UI/             课 13~14 — 界面
├── Input/          课 15 — 点击操作
├── AI/             课 16 — 电脑
├── Demo/           课 17 — 场景搭建
├── Perks/          课 18~19 — 福利
└── OA/             课 20 — 审批
```

### 1.3 每课固定流程

1. 读教案「本课步骤」  
2. 打开「涉及文件」，读文件顶部 `课 XX` 注释  
3. 实现代码（先写逻辑，再 Play）  
4. 完成「自测清单」  
5. `git commit -m "课XX: …"`（可选但推荐）

## 课 01｜熟悉项目与 Demo 目标

**学习目标：** 知道最终要做什么，不在本课写代码。

**步骤：**

1. 阅读 `plan.md` 第一、二、三章（世界观、咖啡、前线）  
2. 浏览 `Reference/Battle/BattleManager.cs` **仅浏览**，理解模块分工  
3. 画出你心里的三块区域：后方 / 前线 / 敌方后方  

**自测：** 能用自己的话说明「怎样获胜」「咖啡用来干什么」。

## 课 02｜Core：枚举与常量

**涉及文件：**

- `Core/GameEnums.cs`
- `Core/GameConstants.cs`

**学习目标：** 用枚举表达游戏状态；魔法数字收进常量。

**步骤：**

1. 在 `GameEnums.cs` 补全：`JobType`、`Faction`、`BoardRow`、`TurnPhase`、`GameResult`、`PlayerActionMode`  
2. 在 `GameConstants.cs` 定义：`InitialHqBudget=20`、`MaxCoffeeMax=12`、`OpeningHandSize=3` 等  
3. 编译（Unity 自动），Console 无报错  

**自测：**

- [ ] 枚举与 `plan.md` 一致  
- [ ] 常量被其他文件引用时不报错（下课后才会大量引用）  

**参考：** `Reference/Core/`

---

## 课 03｜Data：卡牌 ScriptableObject

**涉及文件：**

- `Data/OfficeCardData.cs`
- `Data/CardCatalog.cs`

**学习目标：** ScriptableObject 配置 + 运行时内置牌库。

**步骤：**

1. `OfficeCardData`：`displayName`、`job`、`hireCost`、`actionCost`、`maxMorale`、`kpi`、`canAdvance`、`isSupport`  
2. `CardCatalog.CreateDemoDeck()`：用 `ScriptableObject.CreateInstance` 做 8 种牌（见 plan 表）  
3. 实现 `Find(cardId)`、`CreateDemoDeck()` 返回 20 张  

**自测：**

- [ ] `CreateDemoDeck().Count == 20`（可在临时测试方法里 Log）  

**参考：** `Reference/Data/`

---

## 课 04｜Cards：卡组与手牌

**涉及文件：**

- `Cards/RuntimeCard.cs`
- `Cards/Deck.cs`
- `Cards/Hand.cs`

**学习目标：** 运行时实例 vs 配置；洗牌与抽牌。

**步骤：**

1. `RuntimeCard` 持有 `OfficeCardData` + 自增 `InstanceId`  
2. `Deck.LoadFromCatalog` + Fisher-Yates `Shuffle` + `Draw`  
3. `Hand`：`Add` / `Remove` / `MaxHandSize`  

**自测：**

- [ ] 洗牌后两次抽牌顺序大概率不同  
- [ ] 手牌满时 `Add` 无效  

**参考：** `Reference/Cards/`

## 课 05｜Resources：部门状态（咖啡 + HQ）

**涉及文件：**

- `Resources/DepartmentState.cs`

**学习目标：** 纯 C# 状态类；`event` 通知 UI。

**步骤：**

1. 字段：`HqBudget`、`CoffeeMax`、`CoffeeCurrent`  
2. 实现：`TrySpendCoffee`、`RefillCoffee`、`IncreaseCoffeeMax`、`TakeHqDamage`  
3. 变更时触发 `OnChanged`  

**自测：**

- [ ] 咖啡不足时 `TrySpendCoffee` 返回 false 且不扣  
- [ ] HQ 不会减到负数  

**参考：** `Reference/Resources/DepartmentState.cs`

## 课 06｜Battlefield：棋盘格子

**涉及文件：**

- `Battlefield/BoardSlot.cs`
- `Battlefield/BoardController.cs`

**学习目标：** 三行棋盘；每行 1 格（MVP）。

**步骤：**

1. `BoardSlot`：`Row`、`Occupant`、`IsEmpty`  
2. `BoardController.Initialize(1)`：按 EnemyBack → Frontline → PlayerBack 建格  
3. 实现 `GetBackSlot(Faction)`、`FrontlineSlot`、`GetAllUnits()`  

**自测：**

- [ ] `Slots.Count == 3`  
- [ ] `GetBackSlot(Player).Row == PlayerBack`  

**参考：** `Reference/Battlefield/BoardController.cs`

## 课 07｜Battlefield：单位与前线

**涉及文件：**

- `Battlefield/UnitEntity.cs`
- `Battlefield/UnitFactory.cs`
- `Battlefield/FrontlineController.cs`

**学习目标：** 单位占格；前线归属判定。

**步骤：**

1. `UnitEntity`：精神值、`HasActedThisTurn`、`TakeDamage`、`EffectiveKpi`  
2. `UnitFactory.Spawn` 一行封装  
3. `FrontlineController.GetOwner()`：前线格上有谁就是谁的  

**自测：**

- [ ] 单位 `TakeDamage` 至 0 触发 `OnDied`  
- [ ] 空前线 `GetOwner()` 为 null  

**参考：** `Reference/Battlefield/`

## 课 08｜Combat：伤害结算

**涉及文件：**

- `Combat/CombatResolver.cs`

**学习目标：** 静态方法做「纯结算」，不改 UI。

**步骤：**

1. `ResolveUnitVsUnit`：双向扣 KPI  
2. `ResolveUnitVsHq`：只扣 HQ  
3. `ResolveRangedAttack`：Manager 后方攻击免反击  

**自测：**

- [ ] 互殴后双方精神值都减少  
- [ ] 打 HQ 不影响攻击者  

**参考：** `Reference/Combat/CombatResolver.cs`

---

## 课 09｜Turn：回合管理

**涉及文件：**

- `Turn/TurnManager.cs`

**学习目标：** 回合状态机；事件 `OnTurnStarted`。

**步骤：**

1. `ActiveFaction` 初始 Player  
2. `EndTurn`：切换 Player/Enemy；回到 Player 时 `TurnNumber++`  
3. 触发 `OnTurnStarted` / `OnTurnEnded`  

**自测：**

- [ ] 连续 EndTurn 四次，`TurnNumber == 2`  

**参考：** `Reference/Turn/TurnManager.cs`

---

## 课 10｜Battle：部署与开局（BattleManager 上）

**涉及文件：**

- `Battle/BattleManager.cs`（Part 1）

**学习目标：** Facade 模式；开局发牌；部署到后方。

**步骤：**

1. 组合：`BoardController`、`TurnManager`、双方 `DepartmentState`、Deck/Hand  
2. `StartBattle`：初始化棋盘、洗牌、抽 3 张、`TurnManager.StartBattle`  
3. `TryPlayCardFromHand`：校验后方空格、扣 Hire Cost、Spawn 单位  
4. 实现 `OnMessage` / `OnStateChanged` 事件  

**自测：**

- [ ] 部署后咖啡减少、手牌减 1（可先 Log，UI 课 13 再接）  

**参考：** `Reference/Battle/BattleManager.cs`（前半）

---

## 课 11｜Battle：推进与攻击

**涉及文件：**

- `Battle/BattleManager.cs`（Part 2）

**步骤：**

1. `TryAdvanceUnit`：扣 Action Cost、移到 Frontline  
2. `TryAttackUnit` / `TryAttackHq`：调用 `CombatResolver`  
3. `CanUnitAttack` / `CanAttackEnemyHq`：Manager 后方规则  
4. 死亡：`HandleUnitDied` 清格子  

**自测：**

- [ ] 推到前线后 `FrontlineController.GetOwner()` 正确  
- [ ] HQ 到 0 时 `Result != Ongoing`  

**参考：** `Reference/Battle/BattleManager.cs`（中段）

---

## 课 12｜Battle：HR、回合结束、胜负

**涉及文件：**

- `Battle/BattleManager.cs`（Part 3）

**步骤：**

1. `BeginTurn`：咖啡上限+1、回满、抽牌、重置 HasActed  
2. `TryHrHeal`：HR 治疗 / 加 KPI  
3. `EndTurn` → 切换回合  
4. `CheckVictory`：HQ ≤ 0  

**自测：**

- [ ] 多回合咖啡上限递增（上限 12）  
- [ ] 结束回合后换边  

**参考：** `Reference/Battle/BattleManager.cs`（后半）
